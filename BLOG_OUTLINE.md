## 博客 1：《我是怎么在 AI 数据分析助手中自实现 8 项统计算法的》

### 1. 背景：为什么统计学硕士的 AI 项目不能只是"调包"？
- 痛点：市面 AI 数据分析产品大多 = "上传 Excel + 调 LLM 直接解读"，结果不可复现、数字不可信。
- 我的立场：AI 项目也要有**方法学深度**。自实现核心算法 = 把统计能力"外化"成代码，而不是藏在黑盒里。
- 钩子金句：「调包能跑通流程，手写才跑通面试。」

### 2. 自实现边界：三层取舍
- 核心计算 NumPy 手写（OLS / K-Means / Welch t / 卡方 / KS / Pearson / IQR+Z-score / RFM）
- 分布 CDF / p 值查表调 SciPy（成熟底层数学库，再造无意义）
- Isolation Forest / PCA 照用 sklearn（工程性价比低）
- 决策原则：**"能体现建模能力的手写，纯工程成熟的复用"**

### 3. 每项算法的一句话原理 + 验证策略
| 算法 | 一句话原理 | 验证策略（对照） |
|------|-----------|----------------|
| OLS | 正规方程 β=(XᵀX)⁻¹Xᵀy，用 solve 求逆更稳 | vs statsmodels：β/R²/se/p 误差 <1e-6 |
| K-Means | k-means++ 初始化 + Lloyd 迭代，轮廓系数选 k | vs sklearn：Iris ARI≥0.7 |
| Welch t | Satterthwaite 近似 df + Cohen's d 效应量 | vs scipy(equal_var=False) |
| 卡方 | 统计量 + Cramér's V，剔除全零行列 | vs scipy.chi2_contingency |
| KS | Lilliefors 临界值（小样本查表插值） | vs scipy.kstest |
| Pearson | 手写协方差 + t 双尾 p | vs scipy.pearsonr |
| IQR/Z-score | 分位距 / 标准化距离判异常 | 与定义一致 |
| RFM | 五分位打分 + 规则分群 | 与手动分群一致 |

- 关键段落示例（OLS）：
  > 「很多人用 `np.linalg.inv(XᵀX)` 直接求逆，但条件数大时数值不稳定。我用 `np.linalg.solve(XᵀX, Xᵀy)` 等价求解，并显式构造 (XᵀX)⁻¹ 来推导标准误——这一步让我在面试里能讲清'为什么协方差矩阵的逆既是系数也是方差来源'。」

### 4. 取舍案例：为什么 Isolation Forest 不手写？
- 投入产出比分析：IF 的核心是随机森林式递归划分，数学成熟；自实现数百行且难超 sklearn 质量。
- 但包装层我自己写（score 语义对齐、阈值参数化、与自研 IQR/Z-score 统一接口），保证"异常检测"模块可替换、可对比。

### 5. 可解释性设计：AnalysisResponse 三件套
- `data`（计算结果）+ `explanation`（方法/假设/解读/局限）+ `meta`（方法名/参数）
- 价值：每个数字都带着"它是怎么来的"一起返回，前端可一键展开公式。
- 结尾金句：「可解释不是 UI 上的装饰，是数据结构的默认值。」

## 博客 2：《LLM 在数据分析产品中的正确角色：翻译器而非算命师》

### 1. 问题：让 LLM 直接分析数据的三重风险
- 幻觉：编出不存在的相关、假百分比
- 不可复现：同样数据两次回答不一致
- 黑盒：无法回溯"这个数字哪来的"
- 真实翻车案例：直接 prompt "分析这份销售数据" → 模型给出具体 GMV 增长 23.7%，但数据里根本没有这一列。

### 2. 本项目的解法：结构化计算 → 注入 prompt → LLM 翻译
- 架构图（文字版）：统计模块产出 JSON → System Prompt 明文约束"数字仅来自注入数据、禁止编造" → LLM 生成自然语言 + 来源标签
- 关键约束代码思想：`assumptions = ["LLM 仅翻译结构化计算结果，所有数字来自前序统计模块", "不编造任何统计量"]`

### 3. 降级策略：LLM 不可用时的结构化摘要
- 触发：未配置 key / 调用超时 / JSON 解析失败
- 行为：回退为分群占比、显著检验、回归摘要等结构化文本，保证"永远有输出"
- 价值：把对第三方 API 的强依赖变成弱依赖，SLA 可控。

### 4. 可溯源设计：每个 AI 结论标注数据来源
- 前端把关键发现里的 `[相关性分析]` / `[回归分析]` 渲染成可点击标签，跳回对应分析页
- 价值：用户能"点进去看原始数字"，信任成本陡降
- 结尾金句：「AI 的信任，来自它敢说'这个数字的出处在这'。」

### 5. （可选）延伸：为什么不做一个 Chat 界面？
- Chat 鼓励自由提问 → 诱发编造；报告聚合模式把 LLM 锁在"已验证数据"上，从源头堵死幻觉。
