# 3 分钟演示脚本（录屏 + 旁白逐字稿）

> 用途：照着这份脚本录一段 **3 分钟** 产品演示视频，放到 README 顶部（见下方「录制后贴哪里」）。
> 目标观众：面试官 / 招聘方。重点不是"功能多"，而是**每一步数字都来自真实计算、AI 只做翻译**。

---

## 一、录制前准备（5 分钟）

1. **启动项目**（二选一）：
   - 最省事：双击仓库根目录的 `start-all.cmd` → 自动开两个窗口（后端 + 前端）。
   - 或分开双击 `start-backend.cmd` / `start-frontend.cmd`。
   - 确认前端窗口出现 `ready`、后端窗口出现 `Uvicorn running on http://0.0.0.0:8000`。
2. 打开浏览器访问 **http://localhost:3000**，停在主页（别急着点）。
3. 准备录屏工具（任选其一，都免费）：
   - **Windows 自带**：按 `Win + G` 打开 Xbox Game Bar → 点「捕获」→「● 开始录制」。零安装，最省事。
   - **Loom**（推荐，浏览器插件）：录完直接生成在线链接，贴 README 即可，不用自己传。
   - **OBS Studio**：功能强，但需先设置，时间紧可跳过。
4. （可选，让报告更惊艳）在 `backend/.env` 填好 `OPENAI_API_KEY`。**不填也能录**——AI 报告会自动降级为结构化摘要，其余功能 100% 正常。

> 演示数据：主页点「一键体验 Demo」会**自动生成 500 行电商示例数据并跳转分析页**，比手动传文件更稳。
> 如果你想演示「上传」动作，也可以把 `demo/orders.csv` 拖进上传页。

## 三、录制小贴士

- **语速放慢**、每切一个页面停顿 1 秒，方便观众看清。
- **鼠标别乱晃**，需要强调时用点击代替划来划去。
- 如果某一步手滑点错，**不用重录**——后期剪掉或重录那 30 秒补上即可。
- 总时长控制在 **2:45–3:15**，太短显得单薄，太长面试官没耐心。
- 报告页若没配 `OPENAI_API_KEY`，口播补一句："这里没填 Key，所以走的是降级摘要；配上 Key 就是完整 AI 自然语言版。"——这反而显得你考虑周全。

## 四、录制后贴哪里（2 分钟）

1. **托管视频**：
   - **Loom**：录完自动有链接，复制即可。
   - **YouTube**：上传时可见性选「不公开」或「私享链接」都行，面试官有链接就能看。
2. **拿到链接后**，把链接填进 `README.md` 顶部的视频位（替换 `DEMO_SCRIPT.md` 里说明的占位，或直接编辑 README 中的视频 URL）。
   - GitHub 会在 README 里**自动嵌入** YouTube / Loom 的裸链接，面试官点开即播。
3. 简历里放仓库链接即可，视频在 README 顶部，面试官点进仓库第一眼就看到。

---

## 五、面试官点不开？排查清单

- **前端白屏 / 接口报错**：Railway 后端是否在运行？`https://<后端>.up.railway.app/health` 应返回 `{"status":"ok"}`。
- **CORS 报错**：Railway 的 `CORS_ORIGINS` 是否包含你的 Vercel 域名（`https://<前端>.vercel.app`）？
- **AI 报告只有结构化摘要**：正常——没配 `OPENAI_API_KEY` 就是降级模式，不是 bug。
- **数据丢失**：Railway 容器文件系统是临时的，重启后示例数据会清空，重新点「一键体验 Demo」即可（生产可挂 Volume）。
